# Карманный лингвист

## Что делает этот бот?

**Карманный лингвист** — бот, предназначенный для поиска значений и синониномов заимствованных слов в русском языке по запросу пользователя.

## Как пользоваться ботом?

Чтобы начать работу с ботом, пользователю достаточно ввести команду ```/start```. После этого бот будет ждать, когда пользователь введёт кириллицей заимствованное слово, значение которого он хочет узнать. Ввод слова регистронезависим. Когда слово будет введено, бот найдёт и выведет его значение, взятое из «Современного словаря иностранных слов». После этого бот предложит пользователю найти синонимы для заимствованного слова и при нажатии на соответствующую кнопку найдёт и выведет соответствующую информацию из «Словаря синонимов».

Чтобы получить краткую справку о возможностях бота, пользователю достаточно ввести команду ```/help```.

## Развёртывание приложения на платформе [Railway](https://railway.com/).

**Шаг 1.** Дополнить pom.xml. Ниже представлено то место, которое у нас изменилось. Этот плагин собирает один исполняемый JAR-файл со всеми зависимостями и указывает, с какого класса начать запуск (pack.Main).

```.xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-shade-plugin</artifactId>
    <version>3.5.0</version>
    <executions>
        <execution>
            <phase>package</phase>
            <goals><goal>shade</goal></goals>
            <configuration>
                <transformers>
                    <transformer implementation="org.apache.maven.plugins.shade.resource.ManifestResourceTransformer">
                        <mainClass>pack.Main</mainClass>
                    </transformer>
                </transformers>
            </configuration>
        </execution>
    </executions>
</plugin>
```

**Шаг 2.** В командной строке набираем mvn clean package, если сборка прошла успешно, значит, бот готов к развёртыванию.

**Шаг 3.** Пушим на GitHub конечную версию кода.

**Шаг 4.** Заходим на [Railway](https://railway.com/). Регистрируемся на GitHub, даём доступ к репозиторию с ботом. Когда репозиторий там появится, выбираем его, переходим на вкладку Variables. Там добавляем две переменные: 

BOT_TOKEN =\
BOT_USERNAME =

После этого проект будет собираться и развёртываться. Если всё прошло успешно, появится зелёная надпись ACTIVE. Если развёртывание не получится, можно будет нажать на вкладку Deploy Logs, посмотреть ошибки и исправить их.
